# 保土ヶ谷ガイア 北斗転生 データ取得フロー

ユーザーが「〇月〇日のデータを取得してください」と依頼した場合に実行するワークフロー。

## 前提条件
- 対象機種: **北斗の拳 転生**（保土ヶ谷ガイア）
- P's CUBEのデータは**当日〜翌朝9時頃まで**閲覧可能
- ユーザーは当日夜（23時以降）または翌朝（9時まで）に依頼する

## 手順

### Step 1: データ取得
ユーザーが指定した日付でデータを取得する。

```powershell
npm run fetch-pscube -- --date=YYYY-MM-DD
```

> 日付はユーザーの指示から判断する。例:
> - 「今日のデータ」→ 本日の日付
> - 「2月28日のデータ」→ `--date=2026-02-28`
> - 「昨日のデータ」→ 昨日の日付

// turbo

### Step 2: データ確認依頼
出力されたMarkdownファイル `data/fetched/YYYY-MM-DD_北斗転生.md` の内容をユーザーに提示し、データの正確性を確認してもらう。

**確認ポイント:**
- 全13台（238〜250）のデータが揃っているか
- BIG/REG/G数に明らかな異常がないか
- 差枚(確定)の手動入力が必要な台がないか

### Step 3: ステータス更新
ユーザーが「OK」と確認した後、Markdownファイルのステータスを更新する。

```
- [ ] データ確認済み  →  - [x] データ確認済み (日時)
```

### Step 4: DB登録
バックアップを自動取得してからDBに登録する。

```powershell
npm run register-fetched -- --file=data/fetched/YYYY-MM-DD_北斗転生.md
```

> このコマンドは自動で以下を実行する:
> 1. `npm run backup` でバックアップ取得
> 2. DBにレコードをupsert
> 3. Markdownのステータスを `[x] DB登録済み` に更新

// turbo

### Step 5: 完了報告
登録結果をユーザーに報告する。

**報告内容:**
- 登録件数
- 合計差枚・プラス台数
- バックアップファイルのパス
